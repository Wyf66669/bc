## 项目说明：简历解析与职位匹配 Demo

本项目按照题目要求的五个模块实现了一个简易的简历解析系统，后端使用 **Flask**，前端为原生 **HTML/CSS/JS**。

### 一、运行环境

- Python 版本建议：3.9+
- 操作系统：Windows / macOS / Linux

安装依赖：

```bash
pip install -r requirements.txt
```

启动后端服务：

```bash
python app.py
```

浏览器打开：`http://127.0.0.1:5000/`

### 二、五个模块对应实现

1. **简历上传与解析（必选）**
   - 前端 `index.html` 使用 `<input type="file">` 上传 **单个 PDF** 简历文件。
   - 后端 `/api/analyze` 接口通过 `pdfplumber` 逐页抽取文本，并做基本清洗（去除多余空白、不可见字符等），支持多页简历。

2. **关键信息提取（必选）**
   - 在 `app.py` 中实现：
     - `extract_basic_info`：提取姓名、电话、邮箱、地址（正则 + 关键词行匹配）。
     - `extract_job_info`：提取求职意向、期望薪资（加分项）。
     - `extract_background_info`：提取工作年限、学历背景、项目经历（加分项）。

3. **简历评分与匹配（必选）**
   - 前端提供职位描述输入框，将文本通过表单一并提交。
   - `compute_match_score`：
     - 对简历文本与职位描述做简单分词（按非中文/字母数字字符切分）；
     - 根据关键字重叠比例计算技能匹配分；
     - 结合工作经验、学历关键词做经验/学历评分；
     - 最终给出 0–100 的综合匹配评分，并返回各子项评分与关键重叠词列表。

4. **结果返回与缓存（必选）**
   - `/api/analyze` 以 **JSON** 形式结构化返回：
     - 原始解析文本 `raw_text`
     - `basic_info`、`job_info`、`background_info`
     - `match`（整体/子项评分）
   - 额外实现了一个基于 `SHA256` 的 **内存级缓存** `_CACHE`：
     - 以「PDF 文件内容 + 职位描述」生成 key；
     - 若重复上传同一份简历和职位描述，直接返回缓存结果（字段 `from_cache: true`）。
   - 如需使用 Redis，可在当前逻辑基础上将 `_CACHE` 替换为 Redis 客户端（留有清晰接口）。

5. **前端页面（必选）**
   - 文件：`static/index.html`, `static/style.css`, `static/main.js`。
   - 功能：
     - 完整覆盖上传 PDF、填写职位描述、发起请求、查看关键信息和评分、查看 JSON 原始结果的流程。
     - 页面采用响应式布局，适配 PC 与窄屏。
   - 部署建议：
     - 将 `static` 下内容复制为一个独立前端工程（或直接用该结构），通过 GitHub Pages / Vercel / Netlify 等托管静态页面；
     - 把表单提交地址中的 `/api/analyze` 换成部署在云端（例如 Render、Railway、云服务器）的 Flask 后端地址。

### 三、如何测试

1. 将若干 PDF 简历文件放入本项目目录中（仅作为测试样本，不需要额外配置）。
2. 启动服务后，在网页中选择其中一个 PDF。
3. 在职位描述区域粘贴招聘 JD（例如某岗位职责与要求）。
4. 点击“开始解析与匹配”，页面将展示：
   - 基本信息 / 求职信息 / 背景信息；
   - 匹配评分与重叠关键词；
   - 完整 JSON 返回结果。

### 四、可拓展点

- 将关键字段抽取逻辑替换为机器学习 / 大模型调用，以提升准确率。
- 使用数据库记录多次解析记录，实现历史查询和统计。
- 使用 Redis 做分布式缓存，支持多实例后端。
