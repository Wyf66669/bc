## 模块五：前端页面

**目标**

- 使用一个前端页面，将上述四个模块的能力串联成 **完整的交互流程**。
- 允许在浏览器中完成：上传简历 → 输入职位描述 → 查看解析结果与匹配评分 → 查看 JSON 原始数据。

**主要代码位置**

- 页面结构：`static/index.html`
  - 上传与职位描述表单：`<form id="analyze-form">`
  - 关键信息展示区：`<div id="key-info">`
  - 匹配评分展示区：`<div id="score-panel">`
  - JSON 原始结果区：`<pre id="json-output">`
- 页面样式：`static/style.css`
  - 卡片布局、两列自适应栅格、按钮与状态提示样式等。
- 页面逻辑：`static/main.js`
  - 监听表单提交事件，构造 `FormData`，向 `/api/analyze` 发送 `POST` 请求。
  - 根据返回的 JSON 渲染关键信息卡片与评分卡片。
  - 在页面底部展示格式化的 JSON 原始结果，方便调试与查看结构化数据。

**部署提示**

- 本项目中，前端通过 Flask 的静态资源功能直接提供。
- 如需部署到 GitHub Pages 等静态托管平台：
  1. 将 `static` 目录中的三个文件复制到独立前端工程；
  2. 修改 `main.js` 中 `fetch("/api/analyze", ...)` 为部署在云端的后端地址（例如 `https://your-backend-domain/api/analyze`）；
  3. 将静态页面推送到 GitHub 仓库，并开启 GitHub Pages 即可在线访问。
