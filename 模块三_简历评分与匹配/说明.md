## 模块三：简历评分与匹配

**目标**

- 对解析后的简历与招聘岗位需求（职位描述文本）进行匹配分析。
- 输出整体匹配分及多维度评分（技能、经验、学历等）。

**主要代码位置**

- `app.py`：
  - `tokenize(text: str) -> set`
  - `compute_match_score(resume_text: str, job_desc: str) -> Dict[str, Any]`
  - `/api/analyze` 中调用 `compute_match_score` 得到 `match` 字段。
- 前端展示：
  - `static/main.js` 中的 `renderScore(result)` 将评分结果渲染到页面。
  - `static/index.html` 中的“匹配评分”卡片区域。

**评分逻辑简要说明**

1. 将简历文本与职位描述分别做简单分词（按非中文/字母数字字符拆分，得到去重后的 token 集合）。
2. 计算两者 token 交集占职位描述 token 总数的比例，作为 **技能匹配分（0–1）**，再映射为 0–100 分。
3. 根据简历中是否出现 “3年以上 / 5年以上 / 主管 / 负责人 / 经理” 等词，对 **工作经验匹配分** 做启发式评分。
4. 根据是否匹配 “博士 / 硕士 / 研究生 / 本科 / 大专” 等关键词，对 **学历匹配分** 做启发式评分。
5. 将三个维度加权求和（例如：技能 60%，经验 25%，学历 15%），得到 **综合匹配评分 overall_score（0–100）**。
6. 同时返回重叠关键词列表 `overlap_keywords`，辅助人工进一步判断。
